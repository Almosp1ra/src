# Unix V6++系统优化实现功能说明

## 1. 去除相对虚实地址映射表

#### 实现效果

移除了相对虚实地址映射表机制，改用 MemoryDescriptor 类中存储的代码段、数据段、只读数据段和栈段的起始虚拟地址、大小等绝对字段进行页表映射。
简化了内存管理逻辑、减少了中间映射层的开销、提高了映射效率，并避免了相对索引的维护复杂性。

#### 修改说明

**include/**
- 修改 `MemoryDescriptor.h`：删除了相对表指针。
  
**proc/**
- 修改 `MemoryDescriptor.cpp`: 直接使用各逻辑段的虚空间首地址和长度字段构造页表，原相对表构造函数现在直接设置这些字段并调用页表构造；移除原有的相对表初始化和使用逻辑，但保留函数声明。
- 修改 `Process.cpp`: 移除了引用相对表的逻辑；修改了 SBreak 函数的语句顺序、避免现有功能下计算逻辑段长度改变量时出错。

## 2. 合并代码段和只读数据段

#### 实现效果

将物理内存中的只读数据段的内容与代码段合并、放在正文段中。
优化了内存布局、减少了段切换开销、提高了安全性和稳定性。

#### 修改说明

**include/**
- 修改 `MemoryDescriptor.h`：新增只读数据段的虚空间首地址和长度字段；优化函数的参数声明。
- 修改 `PEParser.h`：新增只读数据段的虚空间首地址和长度字段。

**pe/**
- 修改 `PEParser.cpp`: 读入程序头时，新增读入只读数据段信息的逻辑；读入可执行文件的段内容时，根据正文段是否共享来单独控制只读数据段的读取过程。
  
**proc/**
- 修改 `ProcessManager.cpp`: 修改进程图像的构造过程，使其正确计算合并后物理空间的逻辑段长度。
- 修改 `MemoryDescriptor.cpp`: 构造页表时，识别出只读数据段的页表项，将其映射到正文段中、代码段之后。
- 修改 `Process.cpp`: 修正了合并后物理空间布局变化、可能导致 SBreak 和 SStack 函数中计算逻辑段长度出错的逻辑。
